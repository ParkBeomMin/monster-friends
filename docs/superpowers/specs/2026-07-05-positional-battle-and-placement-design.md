## 1. 목표
현재는 코드에 하드코딩된 두 팀이 "앞 적을 때리는" 단순 자동 전투다. 이걸 **미티파티식**으로 한 단계 올린다:
1. 유닛을 **보드 위 자리에 배치**하고, **위치가 전투에 영향**을 주게 한다 (앞줄이 맞고, 뒷줄 원거리는 보호받음).
2. 배치를 **UI로** 한다 (유닛 목록 → 자리 선택).
3. 화면을 **반응형**으로 만들어 모바일에서 안 잘리게 한다.

## 2. 설계 결정 (기본값 — 잠금)

### 2.1 보드
- 팀당 **2행(앞줄 front / 뒷줄 back) × 3열 = 최대 6칸**.
- 좌측 = 플레이어(A), 우측 = 상대(B). 대칭.
- 유닛 위치는 `{ row: 'front' | 'back', col: 0 | 1 | 2 }`.

### 2.2 위치 전투 규칙 (결정론 유지)
- **타겟 우선순위는 공격자 역할에 따라 다름:**
  - `tank`, `melee`: 적 **앞줄(front)** 중 col 낮은 순 우선 → 앞줄 전멸 시 뒷줄.
  - `ranged`: 적 **전체**에서 col·row 무관하게 규칙 순서로 타격 (뒷줄 저격 가능) → 뒷줄 배치의 의미 부여.
  - `support`: 이번 유닛에서는 `ranged`와 동일 타겟팅(별도 힐 로직은 이후 유닛). 
- 동일 우선순위 내 정렬: `front`가 `back`보다 먼저, 그다음 `col` 오름차순. **완전 결정론** (랜덤 없음).
- 행동 순서(틱 내): 기존과 동일하게 팀(A→B) → 배치 인덱스 순. 위치는 타겟팅에만 영향.

### 2.3 배치 UI (전투 전 화면)
- **탭-투-플레이스**(모바일 친화): 하단 유닛 목록에서 유닛 탭 → 보드 빈 칸 탭 → 배치. 다시 탭하면 제거/회수.
- 상대(B)는 이번 유닛에서 **고정 데모 스쿼드**로 미리 채워둔다 (편성 대상은 A만).
- **"전투 시작" 버튼** → 배치된 A + 데모 B로 `simulateBattle` 실행 → 결과 재생.
- 최소 1유닛 배치해야 시작 가능.

### 2.4 반응형
- Phaser `Scale.FIT` + `autoCenter`로 논리 해상도(예: 960×540, 16:9 가로 기준)를 화면에 맞춰 스케일.
- 세로 폰에서도 전체가 보이도록(레터박스 허용). 가로 전환 시 꽉 차게.

## 3. 범위

### 3.1 포함 ✅
- 엔진: 유닛에 위치(row/col) 추가 + 역할별 위치 타겟팅.
- 배치 화면(탭-투-플레이스) + 전투 시작.
- 전투 재생 시 앞줄/뒷줄 2행 레이아웃 렌더.
- 반응형 스케일.

### 3.2 제외 ❌ (이후 유닛)
- 지원가 힐 로직, 진영 시너지(다음 유닛), 스테이지 진행, 방치, 지휘관, 펫, 아레나.
- 드래그앤드롭(탭으로 충분), 유닛 보유/획득 시스템(지금은 고정 로스터에서 편성).

## 4. 기술
- 기존 스택 유지 (Vite + TS + Phaser + Vitest).
- 엔진 변경은 **순수 로직 + TDD**, `src/engine/` Phaser 미의존 유지.
- 위치 타겟팅은 기존 `selectTarget`을 **역할·위치 인지 버전으로 확장** (결정론 테스트 필수).

## 5. 성공 기준
- 배치를 바꾸면(예: 탱커 앞 / 궁수 뒤) 전투 결과가 **눈에 띄게 달라진다** (위치가 의미 있음).
- 모바일 세로/가로에서 **화면이 안 잘린다**.
- 엔진 위치 타겟팅이 결정론 테스트로 검증된다.

## 6. 미해결 (이후 결정)
- 지원가/힐러의 아군 대상 로직
- 진영 시너지 수치
- 보유 유닛 풀(획득/뽑기)과 편성의 연결
