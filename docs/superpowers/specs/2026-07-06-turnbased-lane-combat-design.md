# 턴제 레인 전투 (미티파티식) 기획서

- 작성일: 2026-07-06
- 상태: 초안 — **확인 대기** (기존 "자동 전투" 모델을 이 모델로 교체)
- 배경: 사용자 정정 — 미티파티는 (1) 선배치가 아니라 **턴마다 1장씩 배치**, (2) **같은 레인만 공격**, (3) 상대 **히어로 HP**를 깎아 승리.

---

## 1. 목표
방금 만든 "다 배치하고 자동 전투" 모델을 **미티파티식 턴제 레인 전투**로 교체한다. 결정론은 시드 기반 난수로 유지(아레나 재현 대비).

**재활용:** `UnitDef`/역할/진영, 보드·렌더링 개념, 배포 파이프라인, TDD 방식.
**교체:** 전투 흐름(자동→턴제), 타겟팅(가로지르기→같은 레인), 승리 조건(전멸→히어로 HP), 편성(선배치→손패 배치).

## 2. 확정 규칙

### 2.1 보드 (실제 미티파티 화면 기준)
- **4레인(행) × 3칸(앞뒤 깊이)**, 팀당. col 0 = 최전방(중앙 쪽), col 2 = 후방.
- 플레이어(A) 좌측, 상대(B) 우측, 중앙선으로 분리. 레인은 좌우가 마주봄 (A의 레인 r ↔ B의 레인 r).
- **히어로는 보드 밖** 좌/우 끝에 별도 배치(격자 밖).

### 2.2 히어로 / 승리
- 양측 **히어로 HP = 120** (v1 라운드 기본값; 실제 MP는 레벨로 스케일 — 이후 튜닝).
- 유닛이 자기 레인에 **적 유닛이 없으면** 상대 **히어로**를 때림.
- 히어로 HP ≤ 0 → 그 편 패배. 최대 턴 도달 시 히어로 HP 높은 쪽 승(동률 무승부).

### 2.3 덱 / 손패 (시드 기반 결정론)
- 각 편 **덱 8장**(`UnitDef`, 중복 허용), **손패 4장** (실제 화면 기준).
- 시작 시 손패 4장 드로우. 매 턴 시작에 손패가 4 미만이고 덱이 남으면 1장 드로우.
- 셔플·드로우는 **시드 PRNG**로 → 같은 시드면 전개 동일(재현 가능).

### 2.4 턴 흐름
플레이어 선공. 한 턴 =
1. (턴 시작) 손패 채우기 드로우.
2. **배치**: 손패에서 카드 1장 선택 → 내 쪽 **빈 셀(특정 칸)** 선택해 배치. (배치 없이 넘길 수 없음; 손패·덱·빈칸 없으면 스킵)
3. **전투 해결(능동 편)**: 능동 편의 살아있는 유닛 각자 1회 공격 — 같은 레인 **최전방 적 유닛**을 때림. 그 레인에 적 유닛 없으면 **상대 히어로** 타격.
4. 승패 체크 → 상대 턴으로.

### 2.5 상대 AI (결정론)
- 시드 기반 드로우 + 단순 규칙 배치(예: 적 유닛이 가장 적은 레인의 최전방 빈 칸, 동률 시 낮은 인덱스). 완전 결정론.

### 2.6 배치/공격 세부
- 배치는 플레이어가 고른 **특정 빈 셀**에.
- 공격 대상은 같은 레인 적 유닛 중 **최전방(col 낮은) 살아있는** 유닛. 없으면 히어로.
- 역할(role)은 이번 유닛에선 스탯 구분용(탱커=고HP, 근접/원거리=딜). 특수 능력은 이후.

### 2.7 제외되는 실제 화면 요소 (v1 이후)
- **칸 보너스 타일**(초록 젬: +공격/+왕관 등), **히어로 스킬/마나**(우하단 1·3·5), **자동전투 버튼**, 유닛 상태이상 아이콘.

## 3. 범위

### 3.1 포함 ✅
- 순수 결정론 엔진(시드): 전투 상태 생성 + "플레이어 배치 → 능동 전투 → 상대 턴(AI 배치+전투)" 진행 함수 + 이벤트 로그 + 승패.
- 인터랙티브 전투 씬: 손패 카드 탭 → 레인 탭 배치 → 자동 해결 → 상대 턴 → 반복.
- 히어로 HP 표시, 레인/유닛 렌더, 승패 표시. 반응형 유지.

### 3.2 제외 ❌ (이후)
- 유닛 특수 능력/스킬, 진영 시너지, 지휘관, 방치, 아레나(멀티), 덱 편집 UI(지금은 고정 덱).
- 애니메이션 고도화, 드래그.

## 4. 기술
- 기존 스택. 엔진 `src/engine/`는 Phaser 미의존 + **시드 PRNG로 결정론**(전역 `Math.random` 금지 유지, 자체 시드 PRNG 사용).
- 기존 `simulateBattle`/위치 타겟팅은 이 모델로 **대체**(제거 또는 미사용). 새 엔진 모듈로 작성.
- 씬: 기존 `PlacementScene`(선배치)는 제거, 인터랙티브 `BattleScene`으로 통합.

## 5. 성공 기준
- 턴마다 손패에서 1장 배치 → 같은 레인 전투 → 히어로 HP 깎기 → 승패까지 **한 판이 돌아간다**.
- 같은 시드로 두 번 돌리면 **완전히 동일한 전개**(결정론).
- 배치 레인 선택이 결과를 바꾼다(레인 전략이 의미 있음).
- 모바일에서 안 잘림.

## 6. 미해결 (이후)
- 덱 편집/보유 유닛 시스템, 역할별 특수 능력, 히어로 스킬, 밸런스 수치 튜닝.
