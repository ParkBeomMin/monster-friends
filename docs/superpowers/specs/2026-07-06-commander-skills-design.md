## 1. 목표
히어로(=지휘관)에 **스킬 시스템**을 붙여 전략 깊이를 준다. 지휘관마다 스킬 3개가 턴에 따라 순차 해금되고, 다 쓰면 HP를 대가로 쓰는 "결단" 국면에 진입한다.

## 2. 확정 규칙

### 2.1 지휘관 = 히어로
- 보드 밖 좌/우의 히어로가 곧 지휘관. HP(120)·승리조건 그대로.
- 각 팀 지휘관은 스킬 사용 진행도 `skillsUsed`(0..3)와 결단 사용 횟수 `desperations`를 가진다.

### 2.2 스킬 3개 (순차 해금, 탭 사용)
- 스킬 i는 **해금 턴 도달 + 앞 스킬(i-1) 사용 완료** 시 사용 가능. 순서 강제(1→2→3).
- 플레이어는 **자기 턴에** 사용 가능한 스킬을 **버튼 탭**으로 1회 발동(원하는 타이밍). 배치와 별개 액션(같은 턴에 스킬+배치 가능).
- 기본 3종 (양 팀 공통, 숫자 튜닝 가능):

| # | 이름 | 해금 턴 | 대상 | 효과 |
|---|---|---|---|---|
| 1 | 집중포화 | 2 | 지정 레인 | 그 레인의 **적 유닛 전체에 18 피해** (죽으면 death) |
| 2 | 진군나팔 | 4 | 없음 | **내 팀 공격력 보너스 +6** (누적 상태; 현재·미래 유닛 모두 적용) |
| 3 | 최후의일격 | 6 | 없음 | **상대 히어로에 45 피해** (0 되면 승리) |

- 팀 공격력 보너스는 상태 `atkBonus[team]`으로 저장, `resolveCombat`이 `def.attack + atkBonus[team]`로 계산.

### 2.3 결단 (HP 소모 스킬, 3개 소진 후)
- `skillsUsed === 3`이면 **플레이어 자기 턴마다 "결단"** 사용 가능(선택적, 탭).
- 발동: **상대 히어로에 `25 + 15*d` 피해**, 동시에 **내 지휘관 HP `12 + 8*d` 소모** (`d` = 지금까지 결단 사용 횟수, 0부터). → 턴 올라갈수록 위력·대가 ↑.
- 내 HP가 0 이하가 되면 **내가 패배**(자멸 가능). 상대 히어로 0이면 승리. 위험/보상 트레이드오프.
- 결단은 **플레이어 전용**(v1). 상대 AI는 결단 미사용.

### 2.4 상대 AI 지휘관 (결정론)
- 자기 턴에 사용 가능한 스킬이 있으면 사용:
  - 스킬1 대상 레인 = **내(플레이어=A) 유닛이 가장 많은 레인**(동률 낮은 인덱스).
  - 스킬2/3은 대상 없음.
- 결단은 사용하지 않음. 완전 결정론.

### 2.5 턴/사용 타이밍
- 스킬·결단 사용은 **턴을 넘기지 않는다**(배치가 턴을 넘김). 한 턴에 스킬 1회 + 배치 1회 가능.
- 사용 후 즉시 효과 적용 + 승패 체크.

## 3. 범위

### 3.1 포함 ✅
- 지휘관 스킬 상태(`skillsUsed`, `atkBonus`, `desperations`) + 3 스킬 효과 + 결단(엔진, 결정론, TDD).
- AI 지휘관 스킬 사용.
- UI: 스킬 버튼 3개(해금/사용됨/사용가능 상태 표시), 스킬1 레인 타게팅(버튼 탭 → 레인 탭), 결단 버튼, 지휘관 HP 표시.

### 3.2 제외 ❌ (이후)
- 지휘관 종류/선택(여러 지휘관), 스킬 커스터마이즈, 애니메이션, 결단 AI 사용.

## 4. 기술
- 엔진 `src/engine/battle.ts`에 스킬 상태·함수 추가(`usePlayerSkill`, 결단, AI 스킬 통합). 순수·결정론 유지.
- `resolveCombat` 데미지 계산에 `atkBonus` 반영.
- 씬에 스킬 UI 추가(상태에서 다시 그리기 유지).

## 5. 성공 기준
- 2·4·6턴에 스킬이 순차로 열리고, 탭으로 발동되어 효과가 보인다.
- 3개 소진 후 결단이 열리고, 쓰면 내 HP를 깎으며 상대 히어로를 강타(턴↑ 위력↑).
- 상대도 스킬을 쓴다.
- 같은 시드+같은 액션열이면 여전히 결정론.
